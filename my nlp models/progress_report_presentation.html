Phase 1 Baseline

TF-IDF Vectorization & Logistic Regression

⚙️ Methodology

A classical machine learning pipeline serving as our benchmark, using TF-IDF representation with binary relevance:

  • Dual TF-IDF Features: Concatenated word n-grams (range 1-2, max 50k tokens) and character n-grams (range 3-5, max 50k tokens) to capture word structures.
  • Binary Relevance (OneVsRest): Fit 6 independent Logistic Regression models, one for each target label.
  • Regularization: Tuned C parameter (C=5.0) to penalize complexity.
  • Limitation: Ignores text context, semantic similarity, and sequential dependencies.

📈 Baseline Results

Despite simplicity, the n-gram baseline proved highly competitive:

  • Weakest per-class ROC-AUC: threat (0.9083).
  • F1-score was limited by rigid decision thresholds (fixed at 0.5) across heavily imbalanced labels.

Phase 2 Deep Learning

Stacked BiLSTM-BiGRU Neural Network

🧠 Neural Architecture

Designed a recurrent deep network to capture sequential word order and contextual dependencies:

  • Embedding + Dropout: 128-dimensional embedding Layer followed by SpatialDropout1D (0.3) to prevent early co-adaptation.
  • Sequential Recurrence: Sticked Bidirectional LSTM (128 units) feeding into a Bidirectional GRU (64 units) to compile long-range context.
  • Dual Pooling Layer: Concatenates Global Max Pooling and Global Average Pooling to capture both salient triggering phrases and overall context.
  • Regularization: Early Stopping on validation mean ROC-AUC (patience=3) and learning rate decay.

📈 Results & Imbalance Regression

Deep learning boosted classification boundary metrics, but highlighted class sparsity issues:

0.9538
Mean ROC-AUC (+0.0089)
0.4097
Mean F1-Score (-0.0780)
  • F1-Score Collapse: F1 score collapsed to 0.0000 for rare labels: threat and identity_hate.
  • Imbalance Vulnerability: Without pretraining or custom label weighting, the network simply learned to output near-zero predictions for minority classes, causing massive false negatives.

Phase 3 Transformers

Fine-Tuning Pretrained DistilBERT

⚡ Transformer Setup

Leveraged the pretrained 6-layer distilbert-base-uncased architecture (66M parameters) to obtain deep bidirectional context representations:

  • Pretrained Knowledge: Initialized with weights trained on massive book corpuses and Wikipedia, allowing fast downstream adaptation.
  • Differential Learning Rate: Fine-tuned DistilBERT backbone at a slow `2e-5` rate while updating the classifier head at a faster `2e-4` rate.
  • Weighted Loss Optimization: Implemented weighted Binary Cross-Entropy loss to penalize predictions on rare classes, counteracting dataset imbalance.
  • AdamW Optimizer + Warmup: Coupled with weight decay and 10% steps linear warmup. Max sequence length capped at 128.

🏆 State-of-the-Art Results

Fine-tuning DistilBERT yielded substantial improvements across all categories:

0.9796
Mean ROC-AUC (+0.0347)
0.4896

  • Best per-class ROC-AUC: obscene (0.9930) and severe_toxic (0.9908).
  • Class Imbalance Solved: F1 scores for rare classes successfully recovered (e.g. threat F1 recovered to 0.1818, identity_hate to 0.2439) while setting new AUC benchmarks.

    Summary & Conclusions

    Key Insights & Next Steps

    💡 Project Insights

    • Imbalance Mitigation: Traditional recurrent architectures fail on rare labels (F1 dropping to 0.0) without explicit handling. Weighted BCE and pretraining are crucial.
    • Pretraining Advantage: Pretrained transformer models (DistilBERT) generalize effectively on small sets (20k dataset) because they contain rich language statistics from pretraining.
    • Context matters: Sequential architectures (RNNs and Transformers) handle contractions, negations, and complex structures, yielding +0.035 ROC-AUC over n-gram baselines.

    🚀 Next Steps

    • Class imbalance focal loss: Explore focal loss or dynamic threshold tuning instead of a generic 0.5 threshold to optimize per-class F1-scores.
    • RoBERTa & DeBERTa scale-up: Evaluate larger transformer models (e.g. `DeBERTa-v3-base`) for further representation capability.
    • Ensembling: Combine DistilBERT with TF-IDF and BiLSTM-GRU predictions to create a robust model blend.
    • Deployment: Export fine-tuned DistilBERT to ONNX format for efficient, real-time edge or server deployment.
